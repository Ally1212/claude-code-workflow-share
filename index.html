半年实战分享

掌握 副驾驶

这不是一篇“装很多插件就会变强”的经验帖。真正有用的，是把 Claude Code 调成一套系统：启动更快，规则更准，技能更顺手，经验能沉淀，流程能复用。

看核心方法 看技能配置
不是模型越强越够用，而是你的系统有没有跟上。

当权限、规则、技能、记忆、文档、hooks 开始协同，Claude Code 才会从“会回答”变成“会协作”。

Layer 01 权限

让开始做事的成本足够低，思路不要被确认框切碎。

Layer 02 规则

把跨项目、长期稳定的偏好写进全局配置，不要反复提醒。

Layer 03 沉淀

把错误、流程和方法留下来，让下一次协作从更高起点开始。

核心判断

不是插件越多越强，而是让 Claude Code 越来越像你。

真正的差距，不在于你有没有追到最新插件，而在于你有没有把自己的工作习惯、判断标准和复盘结果持续写进系统。 你在训练的，不只是一个工具，而是一个越来越懂你的开发伙伴。

Method / 方法

先把底座搭好，再谈效率。

这套工作流最重要的，不是花哨，而是顺序。先降低摩擦，再统一规则，再建立沉淀机制。 顺序错了，后面加再多插件也只是更复杂。

Chapter 01
快

先把启动速度做到足够快

我会先做一个像 `yolo` 这样的自定义命令，核心不是“激进”，而是避免每次进入 Claude Code 都被重复的权限确认打断。只要你的边界是清楚的，降低交互摩擦本身就是生产力。

目标：把“想到要做”到“开始做”的距离缩短。
结果：思考链路更完整，连续性更强。
Chapter 02
准

把全局规则写短，但写准

`CLAUDE.md` 不应该成为一份冗长说明书。更好的做法是只保留跨项目、长期稳定的偏好： 例如始终中文回答、联网调研优先英文高质量信息源、只写通用编码原则、项目特定规则放项目目录。

建议：尽量控制在一屏到两屏内，别让噪音淹没有效约束。
原则：全局只写共性，项目只写个性。
Chapter 03
借

开始写代码前，先问有没有现成技能

每次要做新事情，我先问的不是“怎么写”，而是“现在有没有合适的 skill”。 比如要接 RevenueCat，就先用 `/find-skills` 找相关能力；设计、测试、支付、部署、性能优化也一样。先借力，再开工。

元技能：`find-skills` 负责找，`create-skill` 负责补空白。
好处：减少重复造轮子，让 Claude Code 更快进入正确语境。
Chapter 04
存

问题解决后，把经验写回系统

真正能拉开长期差距的，不是单次回答质量，而是问题解决后有没有形成沉淀。 我会把踩坑、修复方式和避免建议写入项目记忆；把部署、回滚、环境配置这类会重复发生的流程写进 `docs`； 把更抽象的方法论和学习成果写入本地 Obsidian。

项目记忆：降低同类错误再次发生的概率。
项目文档：把高频流程标准化，下次直接让模型照着执行。
外部笔记：把会话里的价值提炼成长期资产。
Stack / 技能

技能不求多，求顺手，求贴身。

推荐先搭元能力，再按真实高频场景继续扩展。装得再多，如果你不用，它就不是能力，只是库存。

Meta Plugins

先装能帮你继续进化的工具

find-skills 先找能力，再决定要不要自己写。
create-skill 市面上没有合适能力时，补上你自己的工作流。
Context7 调最新文档时更稳，尤其适合技术接入和 API 变化快的场景。
claude-mem 把长期记忆接进来，让经验开始积累，而不是反复丢失。
Personal Commands

我常用的自定义命令与技能

/biji 把当前会话的精华整理成 Obsidian 笔记。
/commit 整理本次变更，生成语义化提交信息并推送。
/learn 拿到一个开源项目后，按意图自动拆解和学习。
/xcodo 直接拉起 Xcode 并做基础构建检查，省掉手动开工程。
Environment / 环境

好的环境，应该让你更专注，而不是更忙。

终端、快捷键、分屏、hooks、worktree，这些都不是重点，重点是它们有没有把你的节奏变顺。 适合自己的保留，不适合自己的就删掉。工作流的终点从来不是复杂，而是自然。

Environment
稳

环境只是服务于思考

像 Kaku 这类终端之所以好，不是因为透明，而是因为它能让你更舒服地展开多个上下文。hooks 也是一样，只有在它确实能自动做掉重复工作时才值得保留。多 worktree 也不是必须项，大任务拆分时它有用，但如果让你更乱，就不要为了“高级”而使用。

使用最强模型，做心里真正想做的产品。

你不是在追一个更会说话的 AI。你是在慢慢打磨一套属于自己的协作系统。 当启动足够快、规则足够清晰、技能足够贴身、经验足够可复用，Claude Code 才会越来越像一个真正可靠的副驾驶。
